# Build Prompt: ianbeatty.com Jekyll Site

## Overview

Build a Jekyll site for GitHub Pages hosting. This is a professional personal website for a computational physicist pivoting into biosecurity research. The site should feel clean, modern, and technically credible without screaming "academic homepage."

## Technical requirements

- Jekyll, compatible with GitHub Pages (use `github-pages` gem)
- Start with the "Minimal" theme as a base, but expect CSS customization
- Responsive design (mobile-friendly)
- Dark/light mode toggle is a nice-to-have but not required for v1
- All pages authored in Markdown
- No JavaScript frameworks; keep it simple
- `CNAME` file for `ianbeatty.com` custom domain

## Local development setup

**Important:** The developer does not have Ruby installed and does not want to install it. All Jekyll building and serving must happen via Docker.

Create a `docker-compose.yml` in the project root:

```yaml
version: '3'
services:
  jekyll:
    image: jekyll/jekyll:latest
    command: jekyll serve --watch --force_polling --livereload
    ports:
      - "4000:4000"
      - "35729:35729"   # livereload
    volumes:
      - .:/srv/jekyll
      - jekyll_bundles:/usr/local/bundle
    environment:
      - JEKYLL_ENV=development
volumes:
  jekyll_bundles: {}
```

The named volume `jekyll_bundles` caches gem installations between container restarts so you don't re-download everything each time.

To serve locally: `docker compose up` then visit `http://localhost:4000`.

Also create a `.gitignore` with at least:
```
_site/
.sass-cache/
.jekyll-cache/
.jekyll-metadata
vendor/
```

Note: The repo was initialized with GitHub's Jekyll `.gitignore` template, which likely covers the above. Verify and supplement if needed rather than overwriting.

## Style guidance

- Clean sans-serif body font (system font stack is fine)
- Generous whitespace, not cramped
- Muted color palette — no bright colors. Subtle accent color for links.
- The landing page headshot should be circular or rounded, not too large
- Section headings should be clear but not heavy/loud
- On the research page, the PER section should feel visually secondary to the Biosecurity section (lighter weight heading, or indented, or similar treatment)
- Footer: just "© 2026 Ian D. Beatty" and maybe a small "Built with Jekyll" note
- No sidebar. No widgets. No blog feed. No social media icons.

## GitHub Pages deployment notes

The repo is `github.com/ibeatty/ibeatty.github.io` (GitHub Pages user site). GitHub Pages deploys from the `main` branch root automatically for user sites.

The `.gitignore` was initialized from GitHub's Jekyll template. The repo has no license file (default copyright applies).

### Phased domain setup

**Phase 1 (initial build and testing):** Do NOT create a `CNAME` file yet. The site will be served at `https://ibeatty.github.io`. Set `_config.yml` to:

```yaml
url: "https://ibeatty.github.io"
baseurl: ""
```

**Phase 2 (after DNS is configured and propagated):** Add a `CNAME` file containing exactly:
```
ianbeatty.com
```
And update `_config.yml` to:
```yaml
url: "https://ianbeatty.com"
baseurl: ""
```

The `README.md` should briefly describe the repo as "Source for ianbeatty.com, built with Jekyll and hosted on GitHub Pages." Nothing elaborate.
